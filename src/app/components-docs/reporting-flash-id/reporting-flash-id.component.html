Guide Reporting Flash ID

REPORTING FLASH ID

Le Module Reporting FLASH ID permet de :

  • Visualiser tous les FP créés à partir de FlashAPP
  • Visualiser les pièces d’identité enregistrées par le client
  • Valider ou rejeter la pièce d’identité

Pour trouver le module Reporting Flash ID, il faut aller dans le menu Reporting

Ensuite veuillez cliquer sur :

  • REPORTING FLASH ID

Une fois que vous cliquez sur le bouton REPORTING FLASH ID la page du Reporting suivante s’affiche, où vous pouvez filtrer par :

  • Pays (Rep. democrat du Congo, Rep du Congo et Ivory coast)
  • Région
  • Ville
  • Statut (Valide, Pending, Rejeter et Sans statut)
  • Période (Date début et date fin)
  • Les informations d’un POS (FP ID, Nom et Prénom) En cliquant sur le bouton Filtrer

Une fois que vous cliquez sur le bouton Filtrer, un tableau s’affiche avec les résultats filtrés

EXPLICATION DES ONGLETS REPORTING

  • ID (ordre de l’identification Flashapp)
  • DATE (la date et heure de la création de FP)
  • FLASH ID (le FP de l’utilisateur)
  • NOMS (est composer du NOM suivis du PRENOM de l’utilisateur)
  • STATUT : C’est le statut d’un FP par rapport à l’action de la conformité :
    • Sans statut : le compte est crée, mais l’utilisateur n’a pas encore enregistré sa pièce d’identité
    • Pending : l’utilisateur à enregistrer sa pièce d’identité et est en attente de l’action de la conformité
    • Valider : lorsque la compliance valide la pièce d’identité de l’utilisateur
    • Rejeter : lorsque la compliance rejette la pièce enregistrer par l’utilisateur

      • NB: le statut OFF ( POS désactiver)

  • PIECE ID : le statut Ok | pas de pièce
    • Pas de pièce : c’est lorsque la pièce d’identité n’est pas encore enregistré par l’utilisateur
    • OK : c’est lorsque la pièce d’identité est enregistrée par le client et/ ou vérifiée et peut subir une action
  • ACTION : l'action à faire après vérification
    • Visualiser

Pour exporter le reporting en excel :

  • Cliquer sur le bouton exporter
  • Nommer le fichier du reporting
  • Et valider avec le bouton "OK"
